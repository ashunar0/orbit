# Orbit — 品質管理ガイド

Orbit 全パッケージ（Router / Query / Form）に共通する品質管理の方針。
AI 生成コードの品質・セキュリティ・理解負債をコントロールするためのプロセス。

## 毎回やること（実装のあと）

- [ ] `feature-dev:code-reviewer` でレビューを回す
- [ ] テストを書く（品質保証 + 仕様書としての役割）
- [ ] 指摘が出たら同セッション内で修正 → 必要なら再レビュー

## 新機能が入ったとき

- [ ] **コーチモードでコードを一緒に読む** — 書いた直後が理解コスト最小。後回しにすると理解負債が固定化する
- [ ] **競合の「痛みの歴史」調査** — 同ジャンルのライブラリの GitHub Issues `bug` ラベルから、自分たちにも起きうるバグを洗い出す
  - Router → React Router / TanStack Router
  - Query → TanStack Query / SWR
  - Form → React Hook Form
- [ ] **edge case テスト追加** — 競合調査で見つかった地雷に対するテスト

## 公開・発信の前

- [ ] `/simplify` でコード整理
- [ ] `pnpm audit` で依存の脆弱性チェック
- [ ] README / ドキュメント最終確認

## 継続的に

- [ ] **Dependabot** を GitHub で有効化（依存の脆弱性自動監視）
- [ ] `pnpm audit` を月1くらいで手動確認

## 今やらないこと

| やらない                       | 理由                                       |
| ------------------------------ | ------------------------------------------ |
| 競合のソースコード全読み       | Issue だけで十分。引きずられるリスクもある |
| パフォーマンス最適化           | 遅いと感じてからやる（YAGNI）              |
| SSR セキュリティ対策           | CSR only の間は攻撃面が極小                |
| 動いてるパッケージのリファクタ | 次の機能追加時に一緒にやる                 |
